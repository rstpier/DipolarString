How consensus forms

Ground rules

The model holds a strict epistemic posture, and it applies to contributions too. Disagreement is welcome; what isn't is an unverifiable claim presented as settled.

  • GR and QM rule. Any divergence in an already-tested regime is, by construction, a defect of DS — not a prediction.
  • Quantified or nothing. A numerical contribution merges only if verify_all.py stays at 40/40. Code ships with the prose.
  • Declared maturity level. Each statement is filed: Derived, Calibrated, Postulated, or Closed. No silent drift from postulate to result.
  • Consensus = explicit maintainer agreement on a PR, not a popularity vote. An unresolved technical objection blocks the merge.
  • Every version is traceable. Merge → tag → Zenodo version DOI under the concept DOI 10.5281/zenodo.21196098. You always know who changed what, and when.
